# 酒店订单日志页原型

## 页面定位
展示单笔酒店订单的全链路追踪日志，用于辅助排查下单、采购、支付、取消、退款、售后等链路问题，适合客服、运营、采购与售后团队联合排查订单异常。

## 页面入口
由酒店订单列表中单条订单的“日志”按钮触发，不作为独立菜单入口。

## 当前原型版本
- 当前主文件：`index.html`
- 当前形态：1400px 宽度抽屉式日志控制台
- 当前风格：经典实心时间轴 + 高密度信息排版
- 当前数据：页面内嵌静态 mock 数据 `rawLogs`

## 当前页面结构
### 顶部区域
- 抽屉标题：订单日志
- 当前示例订单：杭州西湖万豪酒店
- 示例订单号：`HTL202604228899`
- 顶部 Tabs：
  - 阶段日志（时序追踪）
  - 业务日志（分类筛选）

### 左侧区域
#### 阶段日志视图
- 采用时间轴展示业务流转过程
- 节点展示时间、标题、日志条数、操作人、状态
- 采购任务类节点额外展示采购任务单号与当前阶段
- 支持点击节点联动右侧明细

#### 业务日志视图
- 按业务分类展示日志集合
- 当前分类包括：
  - 试单
  - 下单
  - 采购
  - 售后/取消

### 右侧区域
- 展示当前节点或当前业务分类下的日志明细
- 当前字段包括：
  - 精确时间
  - 阶段（Phase）/ 执行内容
  - 执行结果
  - 操作人
  - JSON 报文展开查看

## 当前能力
- 支持阶段日志视图
- 支持业务日志视图
- 支持抽屉式日志展示
- 支持经典实心时间轴状态展示
- 支持默认自动选中第一个阶段节点
- 支持采购任务单号提取与展示
- 支持当前阶段展示
- 支持日志节点展开查看 JSON 报文
- 支持复制报文内容
- 支持错误日志高亮展示
- 支持成功 / 失败 / 进行中三种状态区分

## 当前数据说明
当前原型通过页面内 `rawLogs` 模拟完整订单链路，覆盖场景包括：
- 试单成功
- 下单成功
- 采购询价
- 采购任务创建
- 采购支付失败
- 采购轮询
- 采购取消
- 订单取消
- 售后任务创建 / 领取 / 处理
- 订单退款
- 人工补单

## 当前实现说明
### 聚合方式
页面当前将日志分为两类：
- 宏观节点：普通订单阶段日志
- 批次节点：采购任务相关日志（`PURCHASE_TASK*`）

### 状态规则
- 存在错误日志：显示失败
- 批次最后阶段仍处于查询：显示进行中
- 无错误且流程完成：显示成功

### 操作人规则
- 聚合节点内去重展示操作人
- 优先展示非“系统”操作人，否则展示“系统”
