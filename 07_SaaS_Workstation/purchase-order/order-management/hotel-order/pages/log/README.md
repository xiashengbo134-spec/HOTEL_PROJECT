# 酒店订单日志页原型

## 页面定位

展示单笔酒店订单的全链路追踪日志，用于辅助排查下单、采购、支付、取消、退款、售后等链路问题。

## 页面入口

当前主要由酒店订单列表中单条订单的“日志”按钮触发，不作为独立菜单入口。

## 当前原型版本

- 当前主文件：`index.html`
- 当前形态：抽屉式日志控制台
- 当前数据：页面内嵌静态 mock 数据 `rawLogs`

## 当前页面结构

### 1. 顶部摘要区

当前不是简单标题栏，而是订单摘要芯片区，包含：

- 系统订单号
- 外部订单号
- 酒店名称
- 房型名称 - 早餐
- 入离时间
- 间夜 / RP
- 酒店聚合 ID tooltip
- 房型聚合 ID tooltip

### 2. 控制区

- 阶段日志
- 业务日志
- 只看订单
- 只看采购单

### 3. 阶段日志视图

当前为手风琴式阶段聚合列表，支持：

- 同阶段日志合并
- 展开 / 收起
- 全部展开 / 全部收起
- 查看单阶段详情
- 查看单条日志报文

### 4. 业务日志视图

当前为左右结构：

- 左侧业务分类
- 右侧该分类结果明细

当前示例分类包括：

- 试单
- 下单
- 采购
- 取消

### 5. 日志报文抽屉

- 标题：日志报文详情
- 支持复制报文
- 支持收起
- 大尺寸 JSON 查看区

## 当前能力

- 支持阶段日志视图
- 支持业务日志视图
- 支持只看订单 / 只看采购单
- 支持阶段合并与展开
- 支持 JSON 报文抽屉查看
- 支持复制报文内容
- 支持成功 / 失败高亮区分

## 当前数据说明

当前原型通过 `rawLogs` 模拟完整订单链路，覆盖场景包括：

- 试单成功
- 下单成功
- 采购询价
- 采购任务创建
- 采购支付失败
- 采购轮询
- 采购取消
- 订单取消
- 售后任务创建 / 领取 / 处理
- 订单退款
- 人工补单
