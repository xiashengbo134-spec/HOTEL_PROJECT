# 酒店订单日志页设计

## 页面定位
用于展示单笔酒店订单的全链路追踪日志，支持按阶段追踪与按业务分类查看，主要服务于异常排查、链路追踪、履约回溯、采购问题定位与售后处理回放。

## 页面入口
来源于酒店订单列表中单条订单的“日志”操作按钮。

## 页面角色
该页面不是独立菜单入口，而是订单列表的扩展操作页，可作为独立日志页面或列表页中的日志抽屉能力存在。

## 当前已落地原型
- 原型文件：`../pages/log/index.html`
- 当前版本：1400px 宽度的抽屉式日志控制台
- 展示方式：经典实心时间轴 + 阶段/业务双视图
- 原型特征：高信息密度、强调时序追踪、适合订单排查场景

## 页面结构说明
### 1. 顶部区域
- 抽屉标题展示页面名称与订单基础标识
- Tabs 切换两种日志视图：
  - 阶段日志（时序追踪）
  - 业务日志（分类筛选）

### 2. 左侧区域
#### 阶段日志视图
左侧采用时间轴形式展示业务流转过程，当前实现特征包括：
- 使用实心状态图标区分成功、失败、进行中
- 每个节点展示时间、标题、日志条数、操作人、状态
- 采购任务类节点支持展示关联单号与当前阶段
- 支持点击节点后联动右侧明细区域

当前页面中的阶段聚合对象包括但不限于：
- VALIDATE（可定检查）
- ORDER_CREATE（创建订单）
- PURCHASE_INQUIRY（采购询价）
- PURCHASE_TASK_CREATE（采购任务创建）
- PURCHASE_TASK_PAY（采购任务支付）
- PURCHASE_TASK_QUERY（采购任务轮询）
- PURCHASE_TASK_CANCEL（采购任务取消）
- PURCHASE_TASK_STATE_CHANGE（采购任务状态变更）
- ORDER_AFTERSALE_CREATE（创建售后任务）
- ORDER_CANCEL（订单取消）
- ORDER_REFUND（订单退款）
- ORDER_AFTERSALE_RECEIVE（领取售后任务）
- ORDER_AFTERSALE_PROCESSED（处理售后任务）

#### 业务日志视图
左侧切换为业务分类列表，当前按业务类型聚合：
- 试单
- 下单
- 采购
- 售后/取消

每个分类展示对应日志数量，点击后右侧展示该业务分类下的全部日志明细。

### 3. 右侧明细区域
右侧采用表格展示当前选中节点或业务分类下的明细记录。

当前表格字段包括：
- 精确时间
- 阶段（Phase）/ 执行内容
- 执行结果
- 操作人
- 展开查看 JSON 报文

其中：
- 精确时间拆分为日期与时间两行展示
- 阶段列同时展示 `phaseName` 与 `phase`
- 错误日志使用红色高亮
- 成功结果使用绿色高亮
- 可通过展开行查看 JSON 报文
- 支持复制报文内容

## 当前原型交互说明
### 阶段日志交互
- 默认进入“阶段日志”Tab
- 默认自动选中第一个阶段节点
- 点击左侧任意阶段节点，右侧刷新该节点对应的明细日志
- 选中状态以高亮边框与背景色表示

### 业务日志交互
- 点击“业务日志”Tab 后，左侧切换为业务分类列表
- 点击某个业务分类后，右侧显示该分类下全部日志明细
- 当前默认选中分类为“采购”

### 报文交互
- 表格行支持展开
- 若该条日志存在 `json` 字段，则显示 JSON 报文区域
- 支持点击“复制内容”按钮复制报文
- 若不存在报文，则展示“该日志节点无附属 JSON 报文”提示

## 当前页面中的聚合逻辑说明
### 1. 底层日志数据
页面当前通过 `rawLogs` 数组承载底层日志数据。

### 2. 阶段聚合规则
当前原型将日志分为两类：
- 宏观节点：非 `PURCHASE_TASK*` 日志
- 批次节点：`PURCHASE_TASK*` 相关日志

### 3. 批次节点识别
采购任务相关日志会尝试从 `content` 或 `result` 中提取采购任务单号，用于同一采购任务批次聚合。

### 4. 状态判定规则
当前状态判定规则包括：
- 存在错误日志：标记为失败
- 批次最后阶段仍为查询类：标记为进行中
- 无错误且流程完成：标记为成功
- 被后续宏观节点切断但无错误：仍标记为成功

### 5. 操作人展示规则
节点操作人按聚合日志去重处理，优先展示非“系统”操作人，否则展示“系统”。

模拟场景覆盖：
- 试单成功
- 下单成功
- 采购询价
- 采购任务创建
- 采购支付失败（无可用支付设备）
- 采购轮询
- 采购取消
- 订单取消
- 售后任务创建/领取/处理
- 订单退款
- 人工补单任务生成
