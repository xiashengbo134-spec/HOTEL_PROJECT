# 酒店订单详情页设计

## 页面定位

酒店订单详情页用于查看单笔订单的完整信息，页面名称统一为：**订单详情**。

虽然页面名称是“订单详情”，但页面内容不仅包含主订单信息，也包含该订单关联的采购任务与采购执行信息，因此它实际上承担的是：

- 查看主订单基础信息
- 查看订单行程与联系人信息
- 查看主单金额与取消规则
- 查看订单关联的采购任务列表
- 查看当前选中采购任务的采购详情
- 支撑业务员进行订单与采购信息对照核查

## 文档说明

本页采用 **“当前已实现 / 已明确规划”** 的方式说明。

- 当前已实现：以你当前页面和确认口径为准
- 已明确规划：已确定要纳入设计，但允许后续按业务继续补充

## 一、当前已实现

### 1. 页面结构

当前详情页结构已明确为三段式布局：

- 左侧：采购任务区
- 右侧上方：订单信息区
- 右侧下方：采购信息区

说明：

- 页面当前**没有**顶部汇总区、Tabs、折叠面板、底部固定操作栏等复杂结构
- 页面主交互是“左侧任务切换，右侧采购信息联动刷新”
- 主订单信息作为订单级固定信息存在，不随采购任务切换而变化

### 2. 页面区块说明

#### 2.1 左侧：采购任务区

左侧区域用于展示该订单下属的采购任务列表，是详情页最核心的任务切换区域。

其业务作用包括：

- 展示当前订单是否已经产生采购任务
- 展示是否发生了重试采购、拆单采购、补单采购等情况
- 让业务员快速切换查看不同采购任务的执行信息

#### 2.2 右侧上方：订单信息区

右上区域展示系统主订单信息，是业务员判断订单原始需求与目标交付内容的主依据。

#### 2.3 右侧下方：采购信息区

右下区域展示当前选中采购任务的采购执行信息，用于与主订单信息进行对照。

### 3. 左侧采购任务区

#### 3.1 是否一定存在采购任务

当前规则已明确：

- 订单**不一定**一定会有采购任务
- 如果规则设置为不创建采购任务，则该区域可能为空

说明：

- 文档中需要明确空态存在的合理性
- 页面不可默认假设每个订单都已有采购任务

#### 3.2 存在多任务的典型场景

当前已确认以下多任务场景真实存在：

- 失败重试采购
- 分单采购
  - 例如拆分日期
  - 例如拆分间数
- 人工补单
- 多渠道比价后切换渠道
  - 例如原采购任务已下单，但后续发现其他渠道更便宜
  - 新渠道重新采购成功后，再取消原采购任务

因此，左侧任务区不是简单的“采购记录列表”，而是一个真实承载多次采购过程的任务面板。

#### 3.3 当前任务卡片字段

当前左侧任务卡片已确认包含：

- 标题：采购任务
- 采购状态
- 采购任务创建时间
- 外部单号
- 系统单号
- 是否拆单采购标签
  - 例如：`拆单采购`

说明：

- 当任务属于拆单场景时，应展示对应标签
- 当前卡片重点是让用户识别“这是一笔什么采购任务、何时创建、当前什么状态、对应哪笔订单”

#### 3.4 左侧任务切换联动规则

当前已确认联动规则为：

- 点击左侧不同采购任务卡片
- 右侧上方订单信息区保持不变
- 右侧下方采购信息区刷新为当前选中任务对应内容

即：采用 **B 方案**：主订单区不变，采购区变化。

### 4. 右侧上方：订单信息区

订单信息区展示的是主订单基准信息。

#### 4.1 当前已确认展示字段

当前订单信息区已确认展示以下内容：

- 酒店名称
- 酒店地址
- 城市
- 电话
- 酒店 ID 信息
  - 聚合 ID
  - 销售 ID
  - 平台 ID
- 入住人
- 联系人
  - 联系人名称
  - 联系人电话
- 房间名称 - 早餐信息
- 入离时间
- 间夜
- 取消规则
  - 指系统订单的取消规则
- 支付金额
- 结算金额
- 订单来源
- 预订时间
- 支付时间
- 平台订单状态

#### 4.2 ID 信息展示规则

酒店相关 ID 包括：

- 聚合 ID
- 销售 ID
- 平台 ID

当前要求：

- 页面不直接平铺全部 ID
- 通过一个 ID 图标承载
- 鼠标悬浮时展示对应 ID 详情

说明：

- 这是为了控制页面信息密度
- 同时保证管理员有能力查看完整识别信息

#### 4.3 联系人与入住人区分

当前页面需要同时表达：

- 入住人
- 联系人

其中联系人信息包括：

- 联系人名称
- 联系人电话

说明：

- 入住人和联系人不是同一概念，文档中需明确区分

#### 4.4 金额信息展示规则

当前订单信息区中的重点金额字段包括：

- 支付金额
- 结算金额

其展示逻辑与列表页保持一致：

- 页面正文展示支付金额、结算金额
- 鼠标悬浮查看完整财务信息
- 财务悬浮口径与列表页保持一致

说明：

- 支付金额是页面中的核心高亮字段之一
- 结算金额也属于关键财务参考项

### 5. 右侧下方：采购信息区

采购信息区展示当前选中采购任务的执行详情。

#### 5.1 当前已确认展示字段

当前采购信息区已确认包括：

- 采购渠道
  - 例如：携程-12306、平台-应用、艺龙-api
- 采购单号
- 采购订单状态
- 酒店名称
- 酒店地址 / 城市 / 电话
  - 这几项相对次要，建议默认收起或通过展开交互查看
- 入住人
- 联系人
  - 指我方平台采购时填写的联系人姓名 + 联系电话
- 房型信息 - 早餐
- 入离时间
- 间夜
- 采购订单取消规则
- 开票类型
  - 例如：酒店开发票、艺龙开发票等
- 下单账号
  - 如 Trip 邮箱账号、APP 登录手机号等
  - API 下单场景可能不存在该字段
- 已选时段
  - 钟点房场景展示
- 入住时长
  - 钟点房场景展示
- 采购类型
  - 自动采购、手工采购
- 不同采购状态对应的提示信息
  - 例如：重试次数、失败原因等
- 采购价格
- 监控渠道
  - 指我方监控该 PR 的渠道

#### 5.2 酒店补充信息的展示方式

采购信息区中的以下字段不是最核心展示项：

- 地址
- 城市
- 电话

因此建议：

- 默认隐藏
- 通过“展开/收起”方式查看

这类交互虽然不是业务动作按钮，但属于页面信息密度控制的重要交互。

#### 5.3 下单账号展示规则

当前已确认：

- 下单账号需要展示
- 当前使用该页面的角色是订单管理员级别

说明：

- 由于角色具备管理权限，因此可展示较完整的采购执行身份信息
- 若后续角色下放，可再补充权限与脱敏规则

#### 5.4 与主订单金额的视觉关系

当前已确认：

- 不要求把主订单金额与采购价格强制放在同一个区域中对照
- 但需要保证页面视觉上可对齐查看

说明：

- 文档中可写为“强调可对照，不强调强绑定布局”

### 6. 当前详情页交互规则

#### 6.1 核心交互

当前详情页核心交互包括：

- 左侧采购任务切换
- 采购信息区随任务切换刷新
- 次要信息的展开/收起
- ID 图标悬浮查看详情
- 金额信息悬浮查看完整财务口径

#### 6.2 当前未落的业务动作按钮

当前详情页暂时**没有**以下业务动作按钮：

- 采购日志
- 发起售后
- 取消订单
- 重试采购
- 人工补单
- 其他处理动作

说明：

- 日志入口当前仅在列表页存在
- 详情页当前定位仍以“查看与核对信息”为主，而不是“直接操作任务”

### 7. 空态说明

当前详情页需要考虑至少一种空态：

- 订单存在，但未创建采购任务

此时页面建议：

- 左侧采购任务区展示空态说明
- 右侧订单信息正常展示
- 右侧采购信息区根据设计展示空态文案，而非空白区域

## 二、已明确规划

### 1. 采购信息区的按字段类型延展

当前虽然没有底部业务按钮，但你已明确有后续规划：

- 因采购任务模块无法一次展示全部信息
- 未来可能根据字段类型和业务需要增加更多查看入口或按钮

例如：

- 付款信息入口
- 开票信息入口
- 账号信息入口
- 更详细的采购执行信息入口

因此，详情页后续可以逐步从“信息详情页”演变为“信息查看 + 局部扩展入口”的页面。

### 2. 采购信息分层增强

随着字段继续增加，后续可考虑进一步分层：

- 基础采购信息
- 价格与成本信息
- 账号与开票信息
- 失败原因与重试信息
- 钟点房专属信息

### 3. 多任务视图增强

后续左侧任务区如信息复杂度继续增加，可考虑增加：

- 任务类型标签
- 渠道标签
- 当前阶段标签
- 重试次数提示
- 失败原因摘要

## 三、与其他页面关系

### 1. 与列表页关系

- 详情页由列表页单条订单的“订单详情”入口进入
- 详情页承接列表页的下钻查看需求

### 2. 与日志页关系

当前已确认：

- 日志入口只有列表页存在
- 详情页当前没有“采购日志”或“订单日志”入口
- 日志统一通过列表页操作打开侧边弹窗查看

## 五、结论

当前酒店订单详情页本质上是一个“主订单信息 + 多采购任务面板 + 当前采购执行详情”的综合查看页面。

它的核心特征是：

- 页面标题是订单详情，但内容覆盖订单与采购两个维度
- 左侧承接多采购任务切换
- 右上固定展示主订单信息
- 右下联动展示选中任务的采购信息
- 重点服务于订单管理员对订单与采购执行的交叉核对

后续如果继续细化原型说明，可以在此基础上继续补：

- 无采购任务空态
- 拆单采购展示规范
- 钟点房专属字段说明
- 采购失败提示信息样式
- 采购任务卡片状态色规范
- 展开/收起交互说明
