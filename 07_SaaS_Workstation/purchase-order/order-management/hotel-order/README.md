# hotel-order

对应三级菜单：酒店订单。

本模块承接手工订单处理场景下的酒店订单管理，当前已形成三类核心页面能力：

- 订单工作台：`pages/list/`
- 订单详情：`pages/detail/`
- 订单日志：`pages/log/`

## 当前模块定位

当前原型体现的模块能力已经覆盖：

- 订单筛选与任务分流
- 接单 / 拒单等快速处理
- SLA 倒计时识别
- 订单与采购过程核查
- 日志与报文排查

从页面形态上看，这已经不是简单查询模块，而是一个订单处理工作台。

## 当前页面关系

### 1. 列表页

主入口页面，当前重点能力包括：

- 多维筛选
- Tab 任务分流
- 待处理 SLA 倒计时
- 四段式状态展示
  - 系统状态
  - 平台状态
  - 采购状态
  - 售后状态
- 订单详情抽屉
- 日志抽屉

### 2. 详情页

当前详情能力分为两种原型：

- `pages/detail/index.html`
  - 偏独立总览 / 作业页
- `pages/detail/side-drawer.html`
  - 偏列表内快速查看抽屉页

其中主 `index.html` 已体现：

- 订单总览
- SLA 与风险提示
- 采购过程查看
- 金额与状态摘要
- 底部固定动作栏

### 3. 日志页

当前日志页为抽屉式排查台，重点能力包括：

- 顶部订单摘要
- 阶段日志 / 业务日志双视图
- 只看订单 / 只看采购单
- 报文抽屉查看与复制
