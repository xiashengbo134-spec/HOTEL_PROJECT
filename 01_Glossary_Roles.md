# 01_Glossary_&_Roles (全量业务术语与角色定义)
---
### 🟢 I. 八大核心子系统定义 (Core Systems)
- **采集系统 (Collection)**: 负责酒店静态信息抓取，前往各平台渠道采集酒店基本信息、房型定义、RP(Rate Plan)计划。
- **聚合系统 (Aggregation)**: 把采集回的异构信息完成聚合映射，作为我方平台的标准基础信息（畅行酒店）的数据源。
- **报价系统 (Quotation)**: 根据我方设置的加价规则、策略和指标，对全部酒店房型完成各种场景方案的实时报价管理。
- **投放系统 (Injection)**: 将对应酒店房型的报价信息投放至 OTA 平台或我方自营平台售卖，买家购买后由对应租户完成履约。
- **交易系统 (Transaction)**: 核心交易逻辑引擎，负责处理订单的下单、状态变更、正向流转与逆向交易处理。
- **采购系统 (Procurement)**: 订单进入租户管理后台后，由租户发起的向上游供应商执行库存买入和订单确认的系统。
- **自动下单系统 (Auto-Ordering)**: 订单开始采购后，根据配置的客户端、支付宝账号、VCC 账号完成自动支付闭环。
- **售后系统 (After-Sales)**: 负责完成租户订单的退款、取消协商、补缴差价、投诉任务等全流程售后处理。
---
### 🔵 II. 核心业务术语 (Business Glossary)
- **IDaaS**: 统一身份认证中心，实现一次登录、多应用（ERP/SaaS/B2B）访问权限分发。
- **应用 (Application)**: 逻辑业务域单元（如 05_ERP_Admin），决定了菜单和权限的物理边界。
- **租户 (Tenant)**: SaaS 平台的独立运营主体，数据完全隔离，下可挂靠多个签约公司。
- **签约公司 (Signing Company)**: 租户下的法务实体，用于绑定支付账号（支付宝/VCC）及财务对账。
- **RP (Rate Plan)**: 价格计划。规定房型售卖属性（如：含早、取消政策、预付/现付类型、特殊活动）。
- **畅行酒店 (Standard Hotel)**: 聚合系统清洗后的我方唯一标准酒店数据，包含标准 ID 与渠道 ID 的映射。
- **VCC (Virtual Credit Card)**: 虚拟信用卡。自动下单的关键支付手段，含卡号、卡BIN、额度管控等。
- **支付客户端 (Payment Client)**: 承载自动化下单任务的运行节点，监控个人、企业、香港支付宝等存活状态。

### 🟡 III. 全平台角色体系 (Roles Tree)
**【05_ERP_Admin：顶层管控】**
- **系统管理员 (System Admin)**: 负责 IDaaS 用户、应用定义、全局数据字典的创建与维护。
- **运维审计 (Ops Auditor)**: 监控租户初始化状态、应用分发情况，负责全系统安全规则审计。
**【06_ERP_Manager：供应链运营】**
- **信息中心经理 (Info Manager)**: 维护平台档案、城市映射、全局加价规则及核心业务指标。
- **数据治理专家 (Data Steward)**: 监控采集与聚合运行，干预映射关系，维护畅行酒店标准库。
- **接口权限管理员 (ACL Admin)**: 负责 B2B 开放平台 API 密钥、ACL 白名单及交易链路模型配置。
**【07_SaaS_Workstation：租户经营】**
- **租户总控 (Tenant Owner)**: 负责本租户组织架构、公司主体、B2B 买家准入及权限分配。
- **OTA 渠道运营 (Channel Operator)**: 负责飞猪/抖音账号具体经营，操作投放、上下架、修改日历价态。
- **履约监控员 (Order Controller)**: 实时监控交易系统订单转化，处理采购系统异常及手动下单同步。
- **财务总控 (Financial Manager)**: 负责支付宝/VCC 账号绑定、额度动态分配及支付客户端实时监控。
- **售后客服 (CS Representative)**: 负责售后系统任务流，包含退款、取消确认及逆向订单查询。
- **对账会计 (Settlement Accountant)**: 配置下游买家结算规则，生成账单、处理充值及流水核对。
**【08_B2B_Platform：分销采购】**
- **买家运营 (Buyer User)**: 在前台执行酒店搜索、报价对比、订单采购、账户余额充值。
- **买家技术 (Buyer Dev)**: 在开发者管理界面维护 API 凭证、测试 Webhook 并查阅文档。

### 🔴 IV. 跨系统交互逻辑红线 (Logic Redline)
- **数据一致性**: 投放系统和 B2B 交易中心展示的数据必须 100% 来源于“聚合系统”生成的“畅行酒店”。
- **权限继承**: 子应用的角色菜单范围，受控于 ERP 超管在“应用管理”中对租户所属应用的定义。
- **支付校验**: 自动下单前，必须实时调用支付引擎的“额度管理”与“客户端存活”校验接口。
- **逆向闭环**: 售后系统的状态变更必须实时反向同步给交易系统和财务系统，确保资金链条闭环。
